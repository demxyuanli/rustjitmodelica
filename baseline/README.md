# Baseline snapshots

Each run exports under **`baseline/<YYYYMMDD>/`** (date folder name). Typical files: `metrics.json`, `failures_all_lines.txt`, `KEY_LOGS.md`, and focused excerpts.

**JIT validate-perf / `regress-harness jit compare-baseline`:** default baseline JSON (when `--baseline` is omitted) is **`baseline/20260418_three_tier_devloop/jit_perf_baseline.json`**. Scenarios are **`stdlib_bake`** (cold-ish SQLite namespace) vs **`devloop_multi_model`** (warm repeat runs); optional tier thresholds (`std_hit_rate_min_pct`, `user_hit_rate_min_pct`, `aot_hit_rate_min_pct`, `project_rebuild_regression_pct`) are stored for future compare logic. The previous default folder was moved to **`baseline/_legacy/20260417_jit_cranelift_none/`** (same numeric `duration_ms_p50` rows, old scenario ids).

**TestLib full-matrix baseline (optimization A/B):** use **`baseline/testlib_validate_perf_v1/jit_perf_baseline.json`**. It holds **860** rows (`172` TestLib models excluding `negative/` x **5** harness scenarios) captured from `jit validate-perf` with default lib paths (`jit-compiler` + `jit-compiler/TestLib`), `--validation-mode full`, `--set-env RUSTMODLICA_CRANELIFT_OPT_LEVEL=none`, and `--purge-scenario-caches`. Compare thresholds keep **duration** / **codegen_wall** regression caps only; **`speedup_min_ratio` is 0** (analyze runs do not enforce cold/hot speedup), and **tier hit-rate floors are omitted** so `compare-baseline` exits **0** when re-run matches this snapshot. After code changes, re-run the same `jit validate-perf` command, then:

`regress-harness jit compare-baseline --report build/<your_out>/report.json --baseline baseline/testlib_validate_perf_v1/jit_perf_baseline.json`

To refresh numbers in place: `jit update-baseline --report ... --output baseline/testlib_validate_perf_v1/jit_perf_baseline.json --confirm` (then re-apply the small `thresholds` block above if the tool overwrites them from an older default file).

For new captures under **`baseline/`**, prefer committing only **`report.json`** plus **`jit_perf_baseline.json`** (and optional **`bake_manifest-*.json`** for tier bake reproducibility). Root **`.gitignore`** ignores validate-perf bulk under `baseline/**` (`cache_*` trees, **`*.sqlite*`**, `perf_*.json`, `stdout_*.txt`, `stderr_*.txt`, `dep_graph_*.json`, `cache_stats_*.json`, `run_manifest.json`, tiered **`aot_archive*.bin`**, etc.) so local re-runs do not clutter `git status`.

## Three-tier TestLib / regression baselines (2026-04)

| Tier | Path | Tooling |
|------|------|-----------|
| **T1 Correctness** | `baseline/correctness_testlib_v1/` (`validate_ok.json`, `expect_fail.json`) | `regress-harness jit record-correctness`, `jit compare-correctness` |
| **T2 Large-scale perf** | `baseline/large_scale_jit_validate_perf_v2/jit_perf_baseline.json` | `jit compare-baseline --preset large-scale-v2`, `jit update-baseline --preset large-scale-v2`; `jit validate-perf --default-correctness-models --include-jitstress-probe` extends models when `ModelicaTest/JitStress` exists |
| **T3 Full regression rollup** | `baseline/full_regression_v1/regression_summary.json` | `run_regression.ps1 -RecordBaseline baseline/full_regression_v1`; compare with `-CompareBaseline` / `-CompareBaselineCurrent` |
| **T3 (stability profile)** | `baseline/full_regression_v1_stability/regression_summary.json` | Same as T3 but record with `-DisableNativeAccelForStability` when Windows access violations appear under the default AOT/JIT-cache path |

DIR acceleration defaults in `run_regression.ps1`: **`DirUsePrivateCache`** defaults to **on**, **`RUSTMODLICA_STD_CACHE_ROOT`** / **`RUSTMODLICA_USER_CACHE_ROOT`** are set under `%LOCALAPPDATA%\rustmodlica\std` and `%APPDATA%\rustmodlica\user` before `run_modelica_dir_regression.ps1`, which supports **`-TwoStage`** (analyze gate), **`-PerModelTimeoutSec`**, **`-AnalyzeFirstTimeoutSec`**, and parallel **heartbeat** logs while waiting on workers.

**DIR quarantine (local, not committed):** repeated analyze/sim watchdog or OOM hits can append models to **`build_modelica_dir_regress/local/dir_quarantine.json`**. The repo **`.gitignore`** ignores **`build_modelica_dir_regress/local/`**; default runs skip quarantined names (pass **`-RetryQuarantined`** to `run_modelica_dir_regression.ps1` / **`DirRetryQuarantined`** from `run_regression.ps1` to include them). To share a list across machines, copy a hand-managed JSON into **`baseline/<your_baseline_name>/`** (or document paths only) and point **`QuarantineFile` / `DirQuarantineFile`** at that path.

**`run_regression.ps1` public parameters (downstream / CI):** **`[bool]$CleanCaches=$true`** — before cases run, removes the **full** cache tier set so T3 baselines stay reproducible: project AOT archives (`jit-compiler/build/cache/aot_archive*.bin`), flatten SQLite per scope (`jit-compiler/build/cache/{project,std,user}/cache-*.sqlite{,-shm,-wal}`), `path_hash_index.sqlite`, `stage_epochs.txt` / `ir_schema_epoch.txt`, `cache_miss_agg_v1.json`, `model_hotness_v1.json`, warmup locks, plus the JIT codegen tree **`%LOCALAPPDATA%\rustmodlica\jit-codegen`**, the global **`%LOCALAPPDATA%\rustmodlica\std-cache`** root, and the whole **`%APPDATA%\rustmodlica\user-cache`** root. Leaving any one of these behind historically surfaced as `exit=-1073741819` access violations during simulation even on fresh JIT compiles (stale flat-IR / array-size rows survived across binary rebuilds). Pass **`-CleanCaches:$false`** from an interactive PowerShell session to keep caches; for **`Start-Process -ArgumentList`** or other argv-style spawns, prefer **`-KeepCaches`** (a **switch**, no colon-bool) so the child never skips purge by accident and you avoid **`[bool]` binding errors** on `-CleanCaches:False` / split tokens. Warm-cache runs are protected by an **auto self-heal retry**: on `exit=-1073741819` the script reruns the same case once with `RUSTMODLICA_{JIT_CODEGEN_CACHE,AOT_NATIVE_LOAD,FLATTEN_FULL_CACHE,CACHE_SQLITE}=0` and marks the case `self_healed=true` in the detail trail (treat a non-zero self-heal count as a signal to run with default **`-CleanCaches`** next time). **`DisableNativeAccelForStability`** forces **`RUSTMODLICA_JIT_CODEGEN_CACHE=0`** and **`RUSTMODLICA_AOT_NATIVE_LOAD=0`**. Full parameter list and exit-code guidance are in the script header comment at the top of **`run_regression.ps1`**.

**Warm-cache AV triage (Windows COFF):** set **`RUSTMODLICA_COFF_RELOC_TRACE=1`** (summary) or **`=2`** (per-section relocation counts). Stderr lines are prefixed **`[coff-reloc]`** and distinguish **`jit-disk-object`** (codegen disk `.bin` COFF) vs **`native-reloc`** (any **`load_aot_code_blob`** path: AOT archive **or** sim-bundle native load). **`scripts/repro_coff_reloc_trace.ps1`** runs the same model twice in a row so you can diff load summaries after a warm-cache run. Disk and archive COFF paths apply the **same** section filter (omit `.debug` / `.pdata` / `.xdata`). **`load_aot_code_blob`** merges **`builtin_jit_symbol_ptrs()`** before relocation so traces no longer show **`runtime_syms=0`** for externals-only maps on pure TestLib models.

**Native-tier cross-binary self-invalidation (`jit-codegen/*.bin`, `aot_archive*.bin`):** `cache::build_id::binary_build_id()` (xxh3-128 of the running `rustmodlica.exe`, memoized) is folded into both the codegen cache filename and the AOT archive header.
- **`jit-codegen/<stable_hash>.bin`**: `codegen_host_tag()` appends `binary_build_id()`, changing every `CodegenCacheKey::stable_hash` when the binary is rebuilt. Old `.bin` files become unreachable by lookup (not loaded, not relocated); disk reclamation happens lazily via the existing `enforce_global_budget` / `--cache-gc` paths.
- **`aot_archive-{std,user,project}.bin` + legacy `aot_archive.bin`**: header `compiler_version = "<CARGO_PKG_VERSION>|<binary_build_id>"`. `AotArchive::from_bytes` fails fast with `compiler version mismatch` whenever the fingerprint differs, invalidating the whole archive per tier on the first read by a new binary — upstream of the per-TOC `codegen_key_hash` check. No manual `AOT_ARCHIVE_VERSION` / `CODEGEN_CACHE_VERSION` bump required for dev rebuilds; the archive is rewritten on next compile.
Together with the SQLite parent-hash chain below, all four persistent tiers (flatten IR SQLite, array-size SQLite, JIT codegen, AOT archive) now fail closed across binary rebuilds and only the `self_heal` retry remains for residual runtime AVs that are **not** caused by cache drift (runtime reloc races between warm-loaded AOT blobs and freshly JIT-compiled code).

**SQLite parent-hash chain (cross-binary self-invalidation):** `jit-compiler/src/flatten/cache_sqlite.rs` stamps every `cache_entries` row with a **`parent_hash`** derived from the running `rustmodlica.exe`'s content hash (`cache::build_id::binary_build_id()`), the stage epoch, the cache kind, and the row key. Reads re-derive that hash and treat any mismatch as miss. Two consequences downstream:
1. Rebuilding `rustmodlica` (different `.exe` bytes) automatically invalidates every stored row — no need to bump `stage_epochs.txt` / `CODEGEN_CACHE_VERSION` / `AOT_ARCHIVE_VERSION` by hand for IR logic changes. The first run after a rebuild behaves like `-CleanCaches` for SQLite-backed tiers; subsequent warm runs (same binary) hit as usual.
2. `cache_kind_stats` gains a synthetic `__chain_miss__` row whose `put_count` counts silent invalidations; a large jump after a fresh build is expected (catch-up) and should go quiet on the second run.
CLI / baseline artifacts are unchanged; treat this as an internal safety net that reduces how often you need `-CleanCaches` or the runtime self-heal retry.
